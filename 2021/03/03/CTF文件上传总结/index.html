CTF文件上传总结

Web
CTF
发布日期:   2021-03-03
文章字数:   647

文件上传主要类型：

  1. A 客户端javascript 检测(通常为检测文件扩展名)
  2. B 服务端MIME 类型检测(检测Content-Type 内容)
  3. C 服务端目录路径检测(检测跟path 参数相关的内容)
  4. D 服务端文件扩展名检测(检测跟文件extension 相关的内容)
  5. E 服务端文件内容检测(检测内容是否合法或含有恶意代码

客户端js检测

​ 前端通过js代码检测拓展名是否合法，

​ 判断方法**：打开burp,上传1.php，burp里没有任何数据，就弹出警告框，即在客户端就被拦截。

​ 绕过方法**：禁掉js或者通过burp改包

服务端验证绕过(MIME 类型检测)

​ MIME type 互联网媒体类型，在html的请求中为content-type

​ 服务端upload.php

<?php
if($_FILES['userfile']['type'] != "image/gif") { //检测Content-type
echo "Sorry, we only allow uploading GIF images";
exit;
}

绕过：

​ burp将request包的content-type修改

​ 或者上传正常文件改文件内容

​ 或改文件名后上传抓包再改回

​ 常见文件的MIME类型：

拓展名 MIME类型
gif image/gif
png image/png
jpg image/jpeg
js text/javascript
html/htm text/html

服务器检测绕过（目录路径检测）

​ 上传的数据包中，如果存在path(或者其他名称)等能够操作上传路径的参数，修改该参数配合解析漏洞Get Webshell，该 方法一般asp系统用比较多。
​ 例如path参数为如下“upfile/”，可以尝试修改为“upfile.asp/”或者“upfile/1.asp/”或者“upfile/1.asp;.”，注意观 察返回的文件名。返回的文件名可能为：upfile/1.asp;.201704117886.jpg，满足IIS6.0解析漏洞。

服务端校验（黑名单）

  • 判断后缀名

使用不在黑名单里的后缀名

对抗规则

绕过方法：

  • 大小写绕过
  • 利用黑名单里没有但是能被解析的后缀名例如php,php3,php4,php5,php7,pht,phtml,phps
  • 加空格绕过
  • 加.绕过

配合Apache的.htaccess文件上传解析

  • 分布式配置文件，在不同目录设置配置文件

文件头检验

​ 常见的文件头标志：

文件格式 文件头
JPEG (jpg) FFD8FF
PNG (png) 89504E47
GIF (gif) 47494638
HTML (html) 68746D6C3E
ZIP Archive (zip) 504B0304
RAR Archive (rar) 52617221
Adobe Acrobat (pdf) 255044462D312E
MS Word/Excel (xls.or.doc) D0CF11E0

%00截断

在url中%00表示ascii中的0，表示字符串结束即url中读到%00会认为 读取结束，则可以

上传webshell
